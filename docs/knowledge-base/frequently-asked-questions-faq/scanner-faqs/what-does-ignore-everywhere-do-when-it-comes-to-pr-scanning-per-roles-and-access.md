# What does ignore everywhere do when it comes to PR scanning per roles and access?

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/knowledge-base/frequently-asked-questions-faq/scanner-faqs/what-does-ignore-everywhere-do-when-it-comes-to-pr-scanning-per-roles-and-access.md "Edit this page")

* * *

**When using the option `cycode_<violation category name>_ignore_everywhere` in a PR scan, what impact does the user’s role and data acess have on the action in the SCM (for example, in GitHub)?**

## Answer [Copied!](#answer "Copy link to this section")

The option isn’t linked in any way to the role or account access in Cycode. In this case, the “source of truth” is the SCM for any role-based access control (RBAC), and if your organization prefers, you can simply disable the option.
