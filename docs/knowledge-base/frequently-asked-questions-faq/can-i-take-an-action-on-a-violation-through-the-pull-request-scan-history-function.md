# Can I take an action on a violation through the Pull Request Scan History function?

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/knowledge-base/frequently-asked-questions-faq/can-i-take-an-action-on-a-violation-through-the-pull-request-scan-history-function.md "Edit this page")

Updated on Dec 14, 2025 • Published on Jun 24, 2024

* * *

## Answer [Copied!](#answer "Copy link to this section")

The Pull Request Scan History function enables you to perform a rescan of any repository (if the repository is still available).

Note, however, that any violation detected during a rescan from the history function isn’t actionable (for example, a rescan will not unblock a PR merge); to take an action, you’ll need to go to the push event itself, i.e., the actual commit (visible in the [Violations](/view-and-manage-violations) page).
