# Best practice: Using Cycode workflows to catch events

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/knowledge-base/helpful-tips-and-best-practices/best-practice-using-cycode-workflows-to-catch-events.md "Edit this page")

* * *

Cycode’s **workflows** can be used to create an automated alert when a specific policy is met, which helps you to catch important events to ensure that your organization doesn’t miss them.

A workflow allows users to choose a specific policy and create a workflow that defines what happens when a violation of this policy is created. For example, in leaked code, if Cycode determines that there’s a suspicious public repository that’s related in some way to your organization, the workflow you set up will automatically alert you to this important event through any of the integrated alerting or issue tracking integrations currently available with Cycode.
