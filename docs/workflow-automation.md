## Introduction [Copied!](#introduction "Copy link to this section")

Workflows are available in the Automation view. Use workflows to prioritize data, reduce the level of noise for violations, and customize your data view more easily. You can also automate existing and new processes in your organization.

Note

Classification rules have been deprecated, and all automation activities are now handled as workflows.

Workflows can contain one or more Cycode actions, such as:

*   Apply an automatic fix (if one is available)
    
*   Set the status of a violation
    
*   Set the severity of a violation
    
*   Add a custom label to a violation
    

Note

The above workflows were previously called classification rules.

Workflows can also trigger one or more third-party actions, such as:

*   Send a Slack message to specific channel
    
*   Create a Jira ticket for a violation
    
*   Send an email to the relevant persona
    
*   Use a webhook to create an alert
    

You can create a workflow manually, or use a template from the workflow library. You can also view the workflow execution logs to see if the workflows are functioning as expected.
