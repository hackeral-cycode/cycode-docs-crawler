# Attestations

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/context-intelligence-graph/inventory/build-entities/attestations.md "Edit this page")

Updated on Oct 21, 2024 • Published on Oct 21, 2024

* * *

An attestation entity is a document that attests to the authenticity, integrity, and origin of a software artifact, using essential metadata like the artifact’s source. It then signs the document using cryptographic methods, ensuring the artifact’s trustworthiness and compliance with SLSA standards. This enables stakeholders to verify the artifact’s integrity and traceability in the software supply chain.

![](/assets/build-entities-image-bhreaj0g.png)

## Filters [Copied!](#filters "Copy link to this section")

Attestations can be filtered by the following filters:

*   Source: According to the source code hosting sites with which the attestation is affiliated
    
*   Subject: According to the attestation subject
    
*   Is Signed: According to whether the attestation is signed

The following information is provided in the Attestations table:

**Column name**

**Description**

Location

Subjects
