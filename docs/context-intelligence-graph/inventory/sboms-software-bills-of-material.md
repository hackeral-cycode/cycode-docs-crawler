## Overview [Copied!](#overview "Copy link to this section")

Generating an SBOM is a critical practice for modern security and compliance programs. Maintaining an SBOM allows organizations to:

*   Improve visibility: Create a complete and accurate inventory of all open-source and third-party components in use.
    
*   Accelerate vulnerability management: Quickly identify all affected applications when a new vulnerability is discovered in a specific open-source package.
    
*   Ensure license compliance: Track and manage the obligations associated with open-source licenses.
    
*   Meet regulator requirements: Fulfill legal and contractual obligations that mandate software transparency, such as the U.S. Executive Order on Improving the Nation’s Cybersecurity (14028).
