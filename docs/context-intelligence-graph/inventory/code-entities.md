# Code entities

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/context-intelligence-graph/inventory/code-entities/_index.md "Edit this page")

Updated on Apr 1, 2025 • Published on Oct 21, 2024

* * *

Code entities are the various components or elements of source code that can be tracked, analyzed, and monitored. These entities can include files, functions, classes, and other structures within a codebase. By identifying and managing these code entities, Cycode helps organizations gain visibility into their code, assess risks, and ensure compliance with security policies. This enables teams to protect their intellectual property and maintain the integrity of their software development processes. You can manage the following code entities from the Inventories view:

*   [Repositories](/context-intelligence-graph/inventory/code-entities/repositories)
    
*   [Organizations](/context-intelligence-graph/inventory/code-entities/organizations)
    
*   [Packages](/context-intelligence-graph/inventory/code-entities/packages)
    
*   [Teams](/context-intelligence-graph/inventory/code-entities/teams)
    
*   [Users](/context-intelligence-graph/inventory/code-entities/users)
