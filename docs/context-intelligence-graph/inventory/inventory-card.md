# Inventory card

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/context-intelligence-graph/inventory/inventory-card.md "Edit this page")

Updated on Apr 17, 2025 • Published on Oct 20, 2024

* * *

You can click an item in a table to open a card that includes a minimum of the following three tabs:

*   **Overview** - general details about the inventory item that is specific to the inventory type.
    
*   **Comments** - use the Comments tab to share insights and other information about this item with others in your organization.
    
*   **Graph** - shows a relationship graph where you can see how the item is utilized, and the point of exposure that creates the risk.
    

## Additional tabs [Copied!](#additional-tabs "Copy link to this section")

In addition to the three tabs described above, inventory entities may also have additional tabs.

### Code entities [Copied!](#code-entities "Copy link to this section")

All code entities also have a Violations tab that shows a summary of all the detected violations along with their severity, and the following information:

Item

Description

Policy name

The policy that triggered the violation.

Total violations

Total number of violations that this code entity has matching this policy.

### Build workflows [Copied!](#build-workflows "Copy link to this section")

Build workflows include a Build Policy tab that shows the following details of the workflow policy associated with the build.

Item

Description

Workflow policy

Description of the policy.

Prevent policy

Turn this option on to enable Prevent mode.

Note

This option isn’t currently available for self-hosted runners.

Allowed hosts

You can update the list of allowed hosts.

Allowed IPs

You can update the list of allowed IPs.
