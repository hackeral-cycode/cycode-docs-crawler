## Overview [Copied!](#overview "Copy link to this section")

The Workflows page in the Automation view provides a high-level summary of the automation you currently have implemented in Cycode. You can see general statistics about your workflows in the widgets at the top of the page, and view and manage workflows in the table below.

Note

Classification rules have been deprecated, and all automation activities are now handled as workflows.

## Workflow widgets [Copied!](#workflow-widgets "Copy link to this section")

There are three widgets at the top of the Workflows page that show aggregated data about how the workflows are being used.

Widget

Description

Total

Shows the total number of workflows you have, and how many times they have run.

Top used actions

Shows the most commonly automated actions based on the enabled workflows.

Top used workflows

Shows the most commonly used workflows.

## Workflow table [Copied!](#workflow-table "Copy link to this section")

The workflow table lists all the workflows you have in Cycode, along with details about each workflow. You can enable and disable workflows, export the workflow list in CSV format, and create new workflows.

Tip

You can filter the list of workflows by members that created workflows, and by enabled/disabled status.

The following columns are visible by default.

Column

Description

Name

Name of the workflow.

Actions

Actions triggered by the workflow.

Created by

Member that created the workflow.

Last modified

Date and time the workflow was last changed.

Applied retroactively

Indicates whether the workflow was applied to Cycode events that happened before it was created.

Runs

Total number of times the workflow was triggered.

Enabled

Indicates whether the workflow is currently active.

You can add the following columns by clicking the **Add columns** icon.

Column

Description

Created at

Date and time the workflow was created.

Associated Cycode projects

Shows projects that the workflow applies to.
