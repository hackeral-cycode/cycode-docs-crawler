# Delete an existing integration

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/integrations/delete-an-existing-integration.md "Edit this page")

Updated on Nov 18, 2024 • Published on Sep 4, 2024

* * *

When deleting an existing integration, you can now choose to delete the knowledge graph data and violation data for the integration.

**Supported in all cloud and container artifact registry integrations**:

*   AWS
    
*   GCP
    
*   Azure
    
*   Docker Hub
    
*   JFrog
    

**Supported in the following build integrations**:

*   CircleCI
    
*   Jenkins
    
*   Kubernetes
    

To delete an existing integration, go to **Settings > Integrations**.

1.  Go to the three-dot option menu next to the integration you wish to delete and, select **Delete**. You’ll be asked to confirm the deletion.
    
2.  Select whether to delete the knowledge graph integration related data and/or the integration related violations.
    

![](/assets/25907ec-image.png)

3.  Click **Delete Integration** to confirm.

Preventing inadvertant integration deletions

To ensure that you don’t inadvertantly delete an integration, Cycode provides a mechanism in which the integration remains in a “Pending deletion” state for 24-hours before being deleted.
