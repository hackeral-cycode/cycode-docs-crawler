## Overview [Copied!](#overview "Copy link to this section")

Cycode’s connector with Coverity on Polaris enables pulling SAST vulnerabilities from Coverity so you can view and manage them in Cycode.

Note

This is a one-way integration from the third-party API to Cycode.

## Domain [Copied!](#domain "Copy link to this section")

SAST

### Prerequisites [Copied!](#prerequisites "Copy link to this section")

*   A valid API token generated by a user with role Observer.

### Permissions [Copied!](#permissions "Copy link to this section")

Note

Cycode will only pull projects that the user who provided the API Key has permissions to.

Permissions

Access

Observer

Read

## Setup [Copied!](#setup "Copy link to this section")

How to generate an API token in Coverity:

1.  Log into the Coverity platform.
    
2.  Click your user in the top left.
    
3.  Click **Access Tokens > Create New Token**.
    

![](/assets/7f3bcde-image.png)

4.  Provide an indicative name for your token (for example “Cycode Token”) and click **Save**.

![](/assets/7301a43-image.png)

5.  Copy the newly created token and store it somewhere safe.

## FAQs [Copied!](#faqs "Copy link to this section")

### Under which policies can Coverity’s vulnerabilities can be found? [Copied!](#under-which-policies-can-coveritys-vulnerabilities-can-be-found "Copy link to this section")

Based on the ‘Issue Type’ property in Coverity on Polaris, Cycode creates custom policies. For example, a vulnerability in Coverity that holds the issue type **Unsafe output of data into innerHtml** will generate a new custom policy in Cycode with that name.

![](/assets/75eeed8-image.png)

![](/assets/c45e358-image.png)

Note that these policies can also be easily found by navigating to **Policies**\> Filter by **Policy Label = CoverityPolaris**.

### Are there any correlations with Cycode’s SAST? [Copied!](#are-there-any-correlations-with-cycodes-sast "Copy link to this section")

No. Since in SAST there isn’t an indicator for the exact vulnerability (like CVE), therefore, the correlation between two different sources isn’t available.

### Does Cycode enrich the vulnerabilities with data from other connectors? [Copied!](#does-cycode-enrich-the-vulnerabilities-with-data-from-other-connectors "Copy link to this section")

Yes. If your Project represents the repository name, and the Stream represents the Branch name, Cycode will be able to provide more information on the repository itself. For example, if your repository is publicly exposed, the correlation with Cycode will indicate that and add this to the violation risk score.

### Can Coverity’s vulnerabilities be found in the Context Intelligence Graph? [Copied!](#can-coveritys-vulnerabilities-be-found-in-the-context-intelligence-graph "Copy link to this section")

Yes. You can search for all Coverity on Polaris data in the graph with the following query:

Search for the entity Violation and filter by Security Tool property for CoverityPolaris.

![](/assets/04fb7b7-image.png)
