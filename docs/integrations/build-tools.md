## Overview [Copied!](#overview "Copy link to this section")

Ensuring your software supply chain’s security means covering the process from end-to-end, including in downstream environments, to prevent malicious attacks by malware into the build process. That’s why it’s important to integrate Cycode with your build tools to ensure that they’re included in the Cycode scanning process.

Integrating security into build tools typically involves adding security checks or safeguards to each key stage of the CI/CD pipeline. Cycode integrates with major build tools, which enables you to scan for exposed secrets and other sensitive data.

### Why are Build Tools Candidates for Secrets? [Copied!](#why-are-build-tools-candidates-for-secrets "Copy link to this section")

Secrets in the build systems are essential for creating meaningful workflows that communicate and authenticate with various services, such as cloud providers, artifact registries, package managers, and more. If these secrets are well-encrypted and only revealed inside the specific builds authorized to access them.
