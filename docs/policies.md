## Introduction [Copied!](#introduction "Copy link to this section")

The **Policies** window lists all of the policies in Cycode, with all violations and alerts being grouped in these policies. All non-custom Cycode policies are hard-coded and are based on commonly accepted best practices for managing your code and repositories. Users are able to create custom policies using the Graph view to cover policies not provided by Cycode.

Cycode provides these policies out-of-the-box:

*   [Secret Detection Policies](/scanners/pipeline-security/secrets/secrets-detection-policies/)
    
*   [Leaks Policies](/scanners/pipeline-security/leaks/leaks-policies)
    
*   [SAST Policies](/scanners/ast-application-security-testing/sast-static-application-security-testing/sast-policies/)
    
*   [SCA Policies](/scanners/ast-application-security-testing/sca-software-composition-analysis/sca-policies)
    
*   [IaC Policies](/scanners/ast-application-security-testing/iac-infrastructure-as-code/iac-policies/)
    
*   [CI/CD Security Policies](/scanners/pipeline-security/ci-cd-security/ci-cd-security-policies)
    
*   [Container Security Policies](/scanners/ast-application-security-testing/container-image-security/container-security-policies)
    
*   [Cloud Security Policies](/scanners/cloud-security/cloud-security-policies)
    

Policies are grouped by categories, including the following:

Category Choices

Description

All Policies

List all the policies available in the Cycode tenant

Custom Policies

Policies created with the Context Intelligence Graph

Exclusions

The list of policy exclusions

Secrets Detection

Hard-coded secrets in your public and private resources

Leaks

Content exposure on public repositories

SAST

Searching source code for potential security vulnerabilities

SCA

Detects vulnerabilities and licenses with open-source dependencies

IaC

Security risks resulting from cloud misconfiguration

CI/CD Security

Best practices for CI/CD pipelines, including least privilege, permissions, and vulnerabilities

Container Security

Best practices and vulnerabilities for container security

Cloud Security

Analyze your cloud configurations for best practices

Code

Policies that affect the Code section of the SDLC pipeline

Build

Policies that affect the Build section of the SDLC pipeline

Artifacts

Policies that affect the Artifacts section of the SDLC pipeline

Cloud

Policies that affect the Cloud section of the SDLC pipeline

Dependencies

Policies that affect the Dependencies section of the SDLC pipeline

From the Policies page, you can do the following:

*   [**Manage policies**](/policies/manage-policies) : Determine which policies should be enabled and which severity to attribute to each policy
    
*   [**Create custom policies**](/policies/create-and-manage-custom-policies/) : Create custom policies to cover use cases not provided out-of-the-box
