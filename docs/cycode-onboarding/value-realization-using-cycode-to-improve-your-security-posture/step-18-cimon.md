# Step 18: CIMON

Now that you’re fully up to speed and master of your Cycode platform, let’s take it up a notch using Cimon. Cimon (pronounced “Simon”), is your CI/CD pipeline superhero, a runtime security solution that detects and stops software supply-chain attacks on your pipeline. By utilizing revolutionary eBPF technology, Cimon monitors and mitigates attacks within the kernel, denying access to users’ assets as soon as they arise.

Rather than trying to prevent all entry points for attackers into the build environment, Cimon is here to prevent them from performing these actions. As a result, attackers cannot exfiltrate your sensitive data regardless of how the build was compromised.

Cimon monitors your build process, building the trusted list of applications, processes and network connections that are part of your build process. It can then be set to prevent any deviation from the normal behavior of your build process.

[Learn more about how Cimon operates and how to set it up for success](/scanners/pipeline-security/cimon) .
