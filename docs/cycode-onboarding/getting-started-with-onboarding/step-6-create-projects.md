# Step 6: Create projects

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/cycode-onboarding/getting-started-with-onboarding/step-6-create-projects.md "Edit this page")

Updated on May 26, 2024 • Published on May 23, 2024

* * *

Projects are your way to segregate between the different parts of your organization. A project can represent a Product, Team or Pipeline by defining a group of assets, like repositories or cloud accounts, that represents this project. Every project can have one or multiple users defined as Project Managers of the project with a matching user role, and other users of Member or Viewer roles that are associated with the project. If a user has a role in a project, their access to the data across the platform will be limited to the assets and violations correlated to the project in which they have roles.

Note

Users with Admin or Owner roles will always have access to all projects. Users with Member or Viewer roles that are not associated with any specific project will have access to data from all projects.

## Why set up projects at this stage? [Copied!](#why-set-up-projects-at-this-stage "Copy link to this section")

Cycode projects are used to group assets and violations into smaller groups, allowing users to assign members responsible for those assets to manage those violations. All Cycode Members and Viewers assigned to one or multiple projects will only be allowed to interact and view the assets and violations of their projects, while Cycode Admins and Owners can view all the assets that are integrated with Cycode. By implementing projects at this stage, you can ensure that all the violations detected in your system are then assigned to a specific project, and will then be tracked and handled by members of that project, providing accountability and transparency. You can also use projects to track how different units in your company are doing vis-a-vis your security targets (see [Executive Dashboard](/dashboards/cycode-360) for details).

## Creating projects [Copied!](#creating-projects "Copy link to this section")

1.  Go to the Projects page and click the “New Project” button
2.  Provide the information about the project and choose the assets of the project
3.  (Optional) Choose a specific policy category for the project, if required.
4.  Select Cycode users to associate with the project.

Once a project is defined, you’ll see information about the assets and violations associated with the project in the projects table. Clicking one of the projects will show more information about the project in the project card that will open.

You can edit a project at any time, which will allow you to modify any of the configurations defined in the project creation process, by clicking on the three dots on the right side of the projects table and choosing “edit”.

Cycode recommends defining as many projects as required to cover all organization assets.
