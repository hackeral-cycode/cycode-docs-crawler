# Value realization - Using Cycode to improve your security posture

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/cycode-onboarding/value-realization-using-cycode-to-improve-your-security-posture/_index.md "Edit this page")

Updated on Aug 21, 2024 • Published on Jun 24, 2024

* * *

**So how are you going to improve your security posture even further?**

*   Automate more processes using Cycode workflows, to improve your program efficiency and reduce your mean time to remediate (MTTR).
    
*   Create custom policies: While Cycode provides a wide and growing range of policies, you’re the expert in understanding which policies your company needs. Use Cycode custom policies capabilities as needed.
    
*   Create classification rules to help you to specify which policies should be prioritized for tracking, and which are less relevant and don’t need to be monitored, and to exclude false positives that create unnecessary alerts.
    
*   Enable PR scans across all teams and for all security scanners and increase the security requirements gradually to ensure that you’re covering all the important bases.
    
*   Take your security posture another step to the left and drive higher adoption of Cycode IDE plugin across your R&D teams.
    
*   Integrate Cycode to your CI processes, making sure you integrate and deploy only secure code.
