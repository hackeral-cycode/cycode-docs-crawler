# Step 15: Cycode RIG

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/cycode-onboarding/platform-adoption/step-15-cycode-rig.md "Edit this page")

Updated on Dec 3, 2024 • Published on May 23, 2024

* * *

Cycode’s RIG seamlessly integrates with your technologies, aggregating data from various phases to provide a comprehensive understanding of your development process. By analyzing metadata, user behavior, configurations, policies, violations, vulnerabilities, and more, it breaks down silos between technology phases. Additionally, it incorporates data from scanning engines to enrich insights. Cycode’s RIGKnowledge Graph establishes meaningful connections and relationships within this data, offering a holistic view of the technologies. This enables you to correlate activity between tools and phases, addressing common AppSec silos and enhancing security across your organization.

Here are just some of the use cases you can leverage the RIG for:

*   Obtaining visibility into the entire array of technologies
*   Building custom policies based on a graph query
*   Code to cloud coverage - investigating issues
*   Understanding the problem source to help remediate it
*   Generating reports through the RIG
*   Using RIGDiscovery graph queries in workflows
*   Using the out-of-the-box query library to search for potential violations

_(To learn more, see our best practice documentation, [Focus on… Cycode’s Risk Intelligence Graph](/knowledge-base/focus-on-articles/focus-on-risk-intelligence-graph-rig) .)_
