# Step 13: Enable R&D to use Cycode IDE plugin

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/cycode-onboarding/platform-adoption/step-13-enable-r-d-to-use-cycode-ide-plugin.md "Edit this page")

Updated on May 26, 2024 • Published on May 23, 2024

* * *

As part of your company’s integration of a shift-left mentality, you’ll want to ensure that your R&D are using Cycode’s IDE plugins: Cycode’s [VS Code Extension](/developer-experience/developers/cycode-ide-plugins/cycode-vs-code-extension-and-compatibles-editors) or the Cycode [JetBrains IDE Plugin](/developer-experience/developers/cycode-ide-plugins/cycode-jetbrains-ide-plugin) .

Cycode’s VS Code Extension is a plugin for Microsoft’s Visual Studio Code (VS Code) editor that helps users to adopt a shift-left strategy, as does Cycode’s IntelliJ Platform Plugin is a plugin for JetBrains IDEs (e.g., IntelliJ IDEA, PyCharm, WebStorm). The shift-left strategy could significantly help businesses avoid costly repairs and potential complications down the line.

Our recommendation is to choose a pilot team that will use Cycode’s plugin first. Once the pilot team is on board, you should share their success with other teams in the organization to encourage high adoption and safe coding. The [Focus on controlled-shift left](/knowledge-base/focus-on-articles/focus-on-controlled-shift-left) article provides you with step-by-step instructions on how to make this process effective.
