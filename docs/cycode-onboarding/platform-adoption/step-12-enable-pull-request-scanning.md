# Step 12: Enable Pull Request scanning

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/cycode-onboarding/platform-adoption/step-12-enable-pull-request-scanning.md "Edit this page")

Updated on May 26, 2024 • Published on May 23, 2024

* * *

The [pull request (PR) scan](/developer-experience/developers/pull-request-pr-scanning/pull-request-pr-scanners) is one of the most popular tools available to developers and an integral part of shift-left security. Once your organization integrates Cycode with your source code, you can control which repositories and organizations are monitored by Cycode as part of the pull request/merge request (PR) status check scanners.

The Pull Request Scans History feature is a historical record of the scans performed in the platform, which you can view and then download as a report using the download function. Similarly, the Developer Audit function lets you track the activities of your developers in the Cycode platform to see which pull request (PR) action was taken, who are the users in the Cycode platform, the category and resource, when they last performed an action, and whether that action was reviewed. You can also add [company custom guidelines](/developer-experience/developers/pull-request-pr-scanning/add-company-custom-guidelines-to-pr-scans) to the PR scans for violation remediation.

We recommend enabling the pull request scanning gradually. Refer to the [Focus on controlled-shift left](/knowledge-base/focus-on-articles/focus-on-controlled-shift-left) article for a detailed explanation on how to do it right.
