# Step 14: Use Cycode CLI in your CI process

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/cycode-onboarding/platform-adoption/step-14-use-cycode-cli-in-your-ci-process.md "Edit this page")

Updated on May 26, 2024 • Published on May 23, 2024

* * *

The Cycode Command Line Interface (CLI) is an application you can install on your local machine to scan your locally stored repositories for any secrets or infrastructure as code (Iac) misconfigurations. The CLI can be run on files, commits, and repositories while they are under active development, providing security and productivity benefits to developers.

The Cycode CLI allows your developers to check code for potential vulnerabilities before pushing the code to a remote server or on-premises environment. Having a command line interface gives the developer security ownership by fitting directly within the development workflow.

Here’s a complete example of how to integrate the [Cycode CLI with your GitLab Runner CI](/developer-experience/developers/cycode-cli/integrate-the-cycode-cli-with-gitlab-runner) ; note that you can use this same method to integrate Cycode CLI with other CI systems such as GitHub Actions, Jenkins etc.

There are two use cases for the Cycode CLI:

*   Run the [CLI independently as a developer](/developer-experience/developers/cycode-cli)
*   Run the CLI using the Cycode IDE plugins, which ensure that the code is safe and doesn’t just depend on your developers.

You may also want to check out this best practice article, [Using Cycode’s Developer Toolkit](/developer-experience/developers/using-cycode-s-developer-toolkit) .
