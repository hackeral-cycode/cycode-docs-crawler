## Overview [Copied!](#overview "Copy link to this section")

From the **Members** page, you can manage current members, invite new members, and manage global and project roles for members. To view the list of Cycode members, go to **Settings > Access > Member management > Members**.

You can add members manually to Cycode, either individually or in bulk. For more information, go to [Add members manually to Cycode](/cycode-settings/user-administration/manage-cycode-members/add-members-manually-to-cycode) .

Important

If SCIM-only user provisioning is enabled, manual user provisioning is disabled. Users can only be provisioned from your IdP.

Additionally, you can export a current list of Cycode members in CSV format.

## Member details [Copied!](#member-details "Copy link to this section")

The following table describes the columns in the **Members** page:

Column name

Description

Shown by default

Name

Shows the full name of the member.

Yes

Email

Shows the email address of the member.

Yes

Created at

Shows when the member was added to Cycode.

Yes

Last login

Shows the last time the member logged in to Cycode.

Yes

MFA

Shows whether multi-factor authentication (MFA) is enabled for this member.

Yes

Global role

Shows the global role assigned to the member. You can change the global role.

Yes

Project roles

Shows how many projects the user is associated with. Hover over the icon to see the full list. You can edit the project role.

Yes

Unassociated user

Shows whether this user is orphaned (not associated with any groups or projects).

No

Note

You can filter the Members view to show users that aren’t associated with any project. These users were previously linked to one or more projects but were removed from all of them. You can then reassign these users to groups or projects.

1.  Click the **Add column** icon above the table.
2.  Select the **Unassociated user** column to show it in the table.
3.  Click **Add filter** and select **Unassociated user**.
4.  Select the checkmark to view only unassociated users.
