The following table describe the different details of the violation card overviews for Cloud Security. The actual fields displayed may vary depending on the violation.

Field

Description

Description

Description of the violation; if a remediation (Cycode and/or Company Guidelines) is available, the description will also include a link to the Remediation tab.

Source

SCM source in which the violation was detected

Subcategory

The subcategory associated with the violation

Resource

The resource where the violation was detected

## Cloud Security risk score factors [Copied!](#cloud-security-risk-score-factors "Copy link to this section")

Note

Some of the risk factor names may differ slightly from the names of the fields in violations. The names used in the risk factors are designed to reflect the relevant terminology.

Risk Factor

Description

**Base Impact**

Severity

Severity is based on the potential for unauthorized access of sensitive information, and considers the default policy severity and whether the severity was updated by a workflow.

Business Impact

Project business impact is determined by the importance of the project to the business; the violation’s risk score will be the maximum business impact of all the projects to which it’s associated.

**Likelihood**

Publicly Accessible Resource

Analyzes whether the asset in which the violation is located has public exposure.
