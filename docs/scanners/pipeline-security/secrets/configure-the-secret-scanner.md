## Overview [Copied!](#overview "Copy link to this section")

The secret scanner provides a number of options that can be configured to determine how the scanner behaves when it detects hardcoded secrets. You can set up the following:

*   [Manage secret validity checks](/scanners/pipeline-security/secrets/configure-the-secret-scanner/manage-secret-validity-checks/)
    
    *   [Secret Validation APIs](/scanners/pipeline-security/secrets/configure-the-secret-scanner/manage-secret-validity-checks/validity-check-apis)
*   [Auto resolution](/scanners/pipeline-security/secrets/configure-the-secret-scanner/manage-auto-resolution)
    
*   [Manage secret obfuscation](/scanners/pipeline-security/secrets/configure-the-secret-scanner/manage-secret-obfuscation)
    
*   [Setting File Storage Scanning for Secrets](/integrations/cloud-and-infrastructure-services/aws/)
    
*   [Exclude a secret value](/scanners/pipeline-security/secrets/configure-the-secret-scanner/exclude-a-secret-value)
    
*   [Configure PR scans (advanced scanner settings)](/scanners/pipeline-security/secrets/configure-the-secret-scanner/configure-pr-scans)
    

### How often does Cycode scan? [Copied!](#how-often-does-cycode-scan "Copy link to this section")

In most scans, Cycode uses webhooks, so we’ll detect a new secret immediately.

### Which branches does Cycode scan? [Copied!](#which-branches-does-cycode-scan "Copy link to this section")

Cycode scans all branches for secrets.
