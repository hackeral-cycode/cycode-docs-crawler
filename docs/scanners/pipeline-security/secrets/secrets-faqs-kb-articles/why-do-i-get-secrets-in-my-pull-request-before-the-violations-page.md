# Why do I get secrets in my Pull Request before the Violations page?

## Answer [Copied!](#answer "Copy link to this section")

When Cycode is analyzing code, we prioritize scanning Pull Requests, due to the fact that Pull Request status checks can block the merging of code. This means that the Violations page will take some time to update when compared to any Pull Request scans, but they will eventually sync up and contain the same results.
