## Overview [Copied!](#overview "Copy link to this section")

Cycode AI combines regex detection, entropy analysis, machine learning, and context validation to deliver accurate secret detection results, reducing noise by up to 90% while catching more real secrets.

This provides the following benefits:

*   Reliable detection of both structured and generic secrets.
    
*   Fewer false positives, less noise for developers and AppSec teams.
    
*   Clear prioritization using confidence scores and **Verified by AI** tags.
    
*   Builds developer trust in secret detection results.
    

### Data Privacy and Security [Copied!](#data-privacy-and-security "Copy link to this section")

Cycode AI is built with a privacy-first architecture to ensure the following:

*   Customer data is never used to train or improve AI models.
    
*   Secrets never leave your environment or VPC.
    
*   Detections remain isolated; no data is shared between customers.
    

### How it works [Copied!](#how-it-works "Copy link to this section")

Cycode AI applies a multi-layered validation process to improve accuracy:

Step

Action

Purpose

Regex and rule matching

Broadly detect potential secrets.

Provide broad coverage.

Entropy and context analysis

Evaluate randomness and code context.

Identify likely custom tokens.

Machine learning models

Evaluate if the value is truly a secret.

Filter out false positives.

Confidence scoring

Assigns a probability score (for example, 94%).

Enable prioritization of real secrets.

Known limitation

The Cycode AI model is trained on secrets found in non-customer repositories (code), and applied only to potential violations detected in SCM tools.

## Leverage Cycode AI for violations [Copied!](#leverage-cycode-ai-for-violations "Copy link to this section")

Cycode AI is enabled in the platform by default. Admin users can disable it as follows:

1.  Go to **Settings** > **Admin settings** > **AI features**.
    
2.  Disable the **Enable ML-based features** option.
    

Cycode AI provides visible guidance in the platform as described below, to help with the remediation process.

### AI confidence score [Copied!](#ai-confidence-score "Copy link to this section")

Each generic secret violation includes an AI confidence score, which indicates the likelihood that the finding is a secret. The score is a percentage, and the higher the score the stronger the confidence.

The AI confidence score is displayed in the Secret violation card > Overview tab > summary tiles.

Data privacy

Cycode AI confidence scores are generated by models that run entirely within Cycode’s VPC and are never trained on customer data.

### Verified by AI tag [Copied!](#verified-by-ai-tag "Copy link to this section")

Cycode automatically labels validated findings with a **Verified by AI** tag. The tag is visible in the following places:

*   Violations page - when grouped by **Secret value**, show the **Secret tags** column
    
*   Secret violation card - In the **Overview** tab >**General info** section
    

You can also filter the table in the Violations view by the **Verified by AI** tag.
