# Cimon and your privacy

[](https://gitlab.com/cycodehq/documentation/cycode_docs/-/blob/main/src/content/scanners/pipeline-security/cimon/security-privacy/cimon-and-your-privacy.md "Edit this page")

* * *

At Cycode, we take our users’ privacy seriously. To this end, Cimon collects and uses **only** the information that’s required to deliver you a great and secure experience and to continue and improve this experience. In addition, data that’s no longer necessary is deleted.

Here are the main things you should know about the data Cimon collects.

## Which data isn’t collected by Cimon​? [Copied!](#which-data-isnt-collected-by-cimon "Copy link to this section")

We never forget to balance our goal of improving Cimon through additional data with our users’ privacy rights and security needs. As a security tool, protecting data is what we’re all about!

Therefore, the following data never leaves your environment and is not shared with anyone, including Cimon:

*   Source Code

## Which data does Cimon collect? [Copied!](#which-data-does-cimon-collect "Copy link to this section")

Our tool collects the following types of data to provide its service and to improve and your protect user however, we access only workflow definitions for the following purposes:

*   Identifying whether Cimon is installed
    
*   Automating pull request onboarding
    
*   Cimon version: This helps us to understand which version of our tool you are using, which is essential for troubleshooting and improving our product.
    
*   Metadata about the CI runner environment: This includes information about the environment in which Cimon is running, such as the operating system and system architecture, which helps us to ensure compatibility and to optimize our tool’s performance. Examples of this metadata include OS version, architecture (amd64/arm), and GitHub runner version.
    
*   Configuration used: This includes information about the configuration options you’ve given, such as the rules you’ve chosen to enforce. Examples of this information include allowed IPs, allowed domains, and prevention mode.
    
*   Metadata about the project analyzed by Cimon: This includes information about the project that you’re analyzing using Cimon, such as the repository name, organization name, and workflow name. This information is used to help you to monitor Cimon installations in the platform. Examples of this metadata include organization name, repository name, and workflow name.
    
*   Generated report: The report that has been generated, including information on any pipeline issues identified. Examples include detection status, network connections, and executed processes.
    
*   Provenance document: The document is uploaded to the platform for further analysis and enrichment. Examples include SLSA provenance with the subjects and runner information.
    
*   Error logs: If Cimon encounters issues, Cimon logs will be sent for further analysis.
    

## Use of collected data [Copied!](#use-of-collected-data "Copy link to this section")

We use the data that Cimon collects for the following main purposes:

*   Present you and your authorized users with relevant data, notifications and finding reports in the Cimon platform dashboard, so that you can take appropriate actions.
    
*   Assist you in creating security policies based on findings and analysis: We analyze the data collected by Cimon to help you create security policies that are specific to your project. As a result, friction is reduced and security policies are kept up-to-date.
    
*   Maintain, support, secure, and continue to improve Cimon and our service: We analyze aggregated and de-identified data (which, for the record, does **not** include personal information) collected by our tool to identify trends and patterns that help us support, secure and improve Cimon so it provide better service to you.
    

## Data sharing​ [Copied!](#data-sharing "Copy link to this section")

We use the services of certain third-party services to provide, maintain, support, and protect Cimon, including, for example, cloud hosting, database and log management, support, and usage and performance analytics. All such third-party services are authorized to handle data solely for the purpose of fulfilling their assigned responsibilities and are obligated not to use or disclose it for any other purpose.

We do not sell your data, period. We also do not share it with third parties other than in accordance with and as described in this documentation page and our Privacy Policy.

An up-to-date list of our sub-processors can be shared upon request.
