## Overview [Copied!](#overview "Copy link to this section")

The Settings view serves as the core administrative hub, accessible to owners and administrators, providing centralized control over crucial tasks and configurations. This module encompasses various key functionalities aimed at bolstering organizational security.

## Platform settings [Copied!](#platform-settings "Copy link to this section")

You can manage most platform administrator tasks from the Settings view, which is visible only to users that are assigned owner and admin roles.

### Integrations [Copied!](#integrations "Copy link to this section")

The Integrations area lets you manage the following:

*   Third-party integrations - connect SCMs, build tools, security tools, and more.
    
*   Cycode Broker instances - view and manage your Broker instances
    
*   Rate limits - configure rate limits for domains
    

### Access [Copied!](#access "Copy link to this section")

The Access area lets you manage the following:

*   Member management - configure user provisioning, and manage global and project roles
    
*   SSO - Integrate your IdP with Cycode and configure role mapping for your users
    
*   Service accounts - add and manage accounts for service (non-human) users
    
*   Session timeout - configure how long a session should be inactive before signing out a user and ending the session
    

### Admin settings [Copied!](#admin-settings "Copy link to this section")

The Admin settings area lets you manage the following:

*   AI features - manage the available AI features that support Cycode functionality
    
*   Plan details - add, manage and delete your Cycode tenants. You can also view the plan details for your organization.
    
*   Projects - manage asset collision warnings if you assign assets to multiple projects
    

### Security posture [Copied!](#security-posture "Copy link to this section")

The Security posture settings area lets you manage the following:

*   Risk score - customize your risk score formula
    
*   SLA - configure handling time for violations of each severity level
    

### Scanners [Copied!](#scanners "Copy link to this section")

Manage general scanner settings and individual scanners settings, such as:

*   Allow using a .cycodeignore file to ignore specific paths during scanning
    
*   Configure various scanner settings
    
*   Configure scanner PR settings
    

### Activity [Copied!](#activity "Copy link to this section")

The Activity area lets you view the following:

*   SCM Logs - inspect the raw events Cycode receives from your connected SCM providers (commits, pushes, pull request updates, and more), and confirm that Cycode is receiving them as expected.
    
*   Audit Logs - review the actions taken by the members of your organization on the Cycode platform.
    

### Back office [Copied!](#back-office "Copy link to this section")

Configure and manage on-demand scans.
